# Day XX - 练习标题

## 今日目标

用 30 到 60 分钟跑通一个最小实践任务。今天重点关注：

1. 输入是什么；
2. 中间处理了什么；
3. 输出是什么；
4. 它和 AIGC / 多模态 / 图像生成实习有什么关系。

## 文件说明

- `README.md`：本说明。
- `data.xxx`：样例数据。
- `main.py`：主脚本。
- `notes.md`：可选，总结关键概念、常见错误、面试表达。

## 运行方式

在当天文件夹下运行：

```powershell
python main.py
```

如果依赖外部库，先安装：

```powershell
pip install package-name
```

如果暂时无法安装外部库，优先使用模拟数据和最小代码替代。

## 整体思路

这里不要只写抽象流程，要写到你可以照着执行。默认包含：

1. 先进入当天文件夹：

```powershell
cd D:\1AIGC_daily_practice\dayXX_topic
```

2. 打开样例数据文件，先观察输入结构：

```powershell
notepad data.xxx
```

观察时重点看：一条样例数据长什么样、有哪些字段、字段值是字符串还是数字、是不是列表。

3. 运行主脚本：

```powershell
python main.py
```

4. 对照输出结果和输入数据，判断程序做了哪几步处理：

- 第一步：读取输入文件。
- 第二步：把原始文本或表格变成 Python 对象。
- 第三步：遍历每条数据。
- 第四步：按规则筛选、统计、转换或保存。
- 第五步：打印结果或写入输出文件。

5. 如果运行报错，先按顺序检查：

- 当前终端是不是在当天文件夹下；
- 文件名有没有拼错；
- Python 是否能正常启动；
- 依赖库是否安装；
- 样例数据格式是否被误改坏。

6. 最后做一个小修改，再运行一次，观察输出变化。这个步骤很重要，因为实习里的代码阅读不是只看懂，而是要能确认“我改了这里，行为就变了那里”。

## 重点理解

这里不只列知识点，要写成“知识点 + 为什么重要 + 工程联系”。例如：

1. 知识点 A：它在代码中出现在哪里。

   详细解释它解决了什么问题。说明如果不用它，代码会变得怎样。再联系 AIGC / 多模态工程，比如数据集配置、prompt 列表、图片路径清单、实验参数配置等。

2. 知识点 B：它在输入到输出的流程中处在哪一步。

   详细解释这一步如何把原始数据变成后续算法能处理的形式。工程里很多模型训练、推理、评测脚本都不是直接处理“看起来的人类材料”，而是先把它们整理成标准结构。

3. 知识点 C：它如何帮助调试和修改。

   详细解释你应该观察哪一行输出、改哪个参数、预期结果会怎样变化。这能训练“读代码 -> 改代码 -> 验证行为”的能力。

## 自查清单

- 问：输入数据的结构是什么？
  答：待补充。要具体说清楚它是 list、dict、csv 表格、文本行，还是图片路径列表。

- 问：代码中最关键的 2 到 3 行是什么？
  答：待补充。要写出具体代码片段，并解释它负责读取、转换、筛选、统计或输出中的哪一步。

- 问：输出结果为什么长这样？
  答：待补充。要把输出中的一项和输入数据中的原始样例对应起来。

- 问：如果我修改一个输入或参数，输出会发生什么变化？
  答：待补充。要给出一个具体例子，比如新增一条数据、改变阈值、改变关键词、改变文件路径。

## 可修改的小练习

1. 修改一个输入样例，观察输出变化。

   指引：先找到样例数据中的一条记录，只改一个字段。运行脚本前先预测输出会不会变。

   参考解法：待补充。写出应该改哪个文件、哪一项、改成什么，以及运行后应该看到什么。

2. 修改一个筛选条件或参数，观察输出变化。

   指引：找到脚本顶部或主流程中的参数，比如关键词、阈值、文件名、最大数量。只改一个参数，避免不知道是哪处改动影响了结果。

   参考解法：待补充。写出应该改哪一行，以及输出数量或内容会如何变化。

3. 新增一条样例数据，确认脚本能正常处理。

   指引：复制已有样例，再改标题、标签、数值或路径。注意保持数据格式合法。

   参考解法：待补充。写出新增样例，并说明它应该被保留、过滤、统计或输出的原因。
