# AIGC / 多模态算法实习每日实践

这个项目负责把每日学习推送里的“今日最小实践任务”落地成可运行的小练习。重点不是一次写大项目，而是每天跑通一个很小的流程，读懂输入、处理、输出和它在 AIGC / 多模态 / 图像生成实习中的意义。

## 学习方式

每一天一个文件夹，例如：

- `day01_python_json`
- `day02_python_path_env`
- `day03_numpy_array`

每个文件夹尽量包含：

- `README.md`：目标、运行方式、详细整体思路、重点理解讲解、自查题和参考答案。
- 样例数据：如 `json`、`csv`、`txt`、图片路径清单等。
- 可运行脚本：代码控制在适合阅读和修改的规模。
- `notes.md`：可选，用来总结关键概念、常见错误、工程联系、面试表达。

## 每日练习流程

1. 先打开当天文件夹的 `README.md`。
2. 按说明运行最小脚本，确认能看到输出。
3. 对照样例数据看清楚输入是什么。
4. 阅读脚本，重点理解中间处理步骤。
5. 修改 2 到 3 个小地方，再运行观察变化。
6. 用自己的话总结今天最重要的 3 个点。

## 创建新练习时的约定

当你从另一个项目复制“今日最小实践任务”过来后，我会优先创建一个新的 `dayXX_topic` 文件夹，并准备：

1. 中文 `README.md`。
2. 最小样例数据。
3. 可运行脚本。
4. 需要时补充 `notes.md`。

之后每天的 `README.md` 默认按下面的要求写：

1. `整体思路` 要具体到操作顺序和运行指令，不只写抽象流程。
2. `重点理解` 不只列知识点，要解释为什么这样写，并尽量联系算法工程、AIGC、多模态或图像生成场景。
3. `自查清单` 要配参考答案，方便你自己对照。
4. `可修改的小练习` 要给对应的指引和参考解法，避免卡在“知道要改但不知道怎么改”。

每次完成后，我会告诉你：文件夹在哪里、创建了哪些文件、先打开哪个文件、运行哪条命令、今天最重要的 3 个理解点。
